# Quietly Labs

Open experiments for small, humane web tools.

Quietly Labs is a public shelf of privacy-aware utilities, BYO-AI templates, calm UI patterns, and retired experiments from the Quietly Tools workshop.

This is not the main Quietly Tools product codebase. The production Quietly Tools app, its private features, backend, analytics, payments, brand assets, and user-data systems are not open source.

## What This Is

Quietly Labs may include:

- GST helper demos.
- BYO-AI writing tools where users bring their own provider key.
- Public library connector demos.
- Calm UI pattern examples.
- Local-first utility experiments.
- Retired or deprecated prototypes rewritten as generic demos.
- Documentation about calm, privacy-first tools.

Everything here should be small, understandable, mock-data-only, and safe for a public MIT repository.

## What This Is Not

Quietly Labs is not:

- The Quietly Tools product source.
- A mirror of production app routes.
- A place for product internals, private backend code, analytics, auth, payments, or donation flows.
- A place for real user content, memoirs, notes, names, emails, photos, coordinates, or financial records.
- A place for premium Quietly Tools illustrations, logos, monk silhouettes, orb visuals, or custom brand assets.

Public code can teach the idea. It must not expose the product implementation.

## Disclaimers

These tools are experimental and provided for learning, prototyping, and convenience.

GST or finance-related utilities are not tax, accounting, or legal advice. Verify important outputs with a qualified professional.

Writing tools, including eulogy or memory-writing helpers, are intended to support your own words and reflection. They do not replace personal judgment, grief support, therapy, or professional care.

BYO-AI examples require your own API key. Review your AI provider's privacy and data-retention policies before sending personal content.

This repository does not include the production Quietly Tools source code, private backend, analytics, payment systems, user data, or proprietary brand assets.

## Boundary Rule

Before adding anything, read [Public Repo Boundaries](docs/public-repo-boundaries.md).

If a file contains production Quietly Tools code, user data, secrets, private URLs, exact production UI, auth, payments, analytics, backend internals, or premium brand assets, it does not belong here.
