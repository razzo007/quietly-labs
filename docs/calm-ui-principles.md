# Calm UI Principles

Calm UI is not sleepy UI. It is interface design that respects the user's attention.

## Principles

- Put the useful thing on the first screen.
- Prefer fewer controls with clearer defaults.
- Let empty space organize the page.
- Use plain labels that describe outcomes.
- Make destructive actions explicit and reversible where possible.
- Keep feedback quiet, specific, and close to the action.
- Avoid fake urgency, streak pressure, and noisy achievement loops.
- Make privacy visible without turning trust into theater.
- Use motion only when it helps orientation or pacing.
- Design for leaving: a good small tool lets the user finish and go.

## Patterns Worth Sharing

- Calm cards with one job each.
- Empty states that help users start, not admire the product.
- Ritual timers with low-pressure completion states.
- Local-first import/export flows.
- Review screens that show what changed before anything is applied.
- Gentle error states with next steps.

## Labs Boundary

Patterns in this repo should be generic teaching versions.

Do not copy exact Quietly Tools production components, layouts, illustrations, orb visuals, monk silhouettes, or branded interaction details.
